# SnapContext 사이드패널 — finalist 완성도 패스

`index.html` 한 화면에서 **finalist 4종의 프로덕션 완성도(craft) 폴리시 버전**을 사이드패널 실폭(약 390px) iframe으로 나란히 비교한다. 폴리시 파일은 `polish/` 하위에 있고, 모든 시안은 단일 HTML·외부리소스 0.

## 완성도 패스 개요

- **정체성은 그대로** — 색·컴포넌트 구조·한국어 카피·시그니처 모티프는 불변. 이번 패스는 미학을 바꾸지 않고 **craft(마감)만 상향**한다.
- **빌드 → 비평 → 픽스 루프** — 각 시안을 프로덕션 완성도로 재제작한 뒤, '흠 없이 출시 가능한가'를 적대적으로 비평(러프엣지 적발)하고, 지목된 흠을 픽스해 다시 검증한다.
- **공통 마감 기준** — 모듈러 타입 스케일 토큰화 + tabular-nums · 4/8 간격 리듬 · 24뷰박스 일관 아이콘 · 4상태(hover/focus-visible/active/disabled) 완비 · 회색 빈 박스 박멸(의미있는 미니 SVG 목업) · idle 무한 애니 금지 · 이모지/× 리터럴 금지 · AA 대비 실측 · reduced-motion 가드 · 신규 마감은 전부 :root 토큰으로(src 이식성 보존).

## 시안별 세부점수 · 핵심 개선 · 남은 흠

세부 8축: 타이포 / 간격 / 아이콘 / 상태 / 깊이·질감 / 콘텐츠 리얼리즘 / 시그니처 일관 / polish (각 10점).

### 1. 영수증·감열지 — craft 8 · 픽스 ✔
- 세부: 타이포 9 / 간격 9 / 아이콘 9 / 상태 8 / 깊이·질감 8 / 콘텐츠 7 / 시그니처 6 / polish 6
- POS 감열지 메타포 — 톱니 절취선·점선 컷라인·도트리더·도트매트릭스 4종 모티프, 복사/공유는 인주 적색 도장 '발행'. 모듈러 타입 스케일(--fs-*, 1.16비율)·tabular-nums·광학 트래킹 일관, WCAG 대비 실측 주석(본문 12.97:1·메타 5.68:1·적색 5.74:1 모두 AA 이상).
- 핵심 개선:
  - 무한 idle 스피너(.spin infinite) 제거 — 명세의 'idle 루프 없음, 도장 1회뿐' 약속과 일치, 업로드중 블록 자체를 삭제.
  - 회색 빈 박스 박멸 — 미리보기는 대시보드 목업, 기록 3행은 대시보드/로그인폼/에러모달 미니어처 SVG로 차별화.
  - 타이포 모듈러 스케일 + tabular-nums 토큰화, 아이콘 24뷰박스·stroke 1.7·round 통일, 4상태 토큰 transition 완비.
- 픽스한 흠: 무한 애니 · 필러 더미 드롭다운(#tpl2) · 업로드중+완료 동시 노출 모순 · 핀 1개+빈상태 공존 · 인라인 매직값/하드코드 hex · 패널에 혼입된 데모 스캐폴딩. (잔존 흠 없음, 검증: infinite·인라인 style 0건)

### 2. 콘크리트 브루탈리즘 — craft 8 · 픽스 ✔
- 세부: 타이포 8 / 간격 9 / 아이콘 9 / 상태 9 / 깊이·질감 9 / 콘텐츠 6 / 시그니처 9 / polish 7
- 노출 콘크리트 슬래브 + 경고 앰버 1점, 라디우스 0 고대비. form-tie·줄눈·스텐실 숫자·하드 오프셋 그림자로 브루탈리즘 정체성 일관. 상태 커버리지 완비, 다이얼로그 포커스 트랩+복귀, reduced-motion 가드.
- 핵심 개선:
  - 타입 스케일 문서/구현 불일치 해소 — 가짜 모듈러 비율 주석 삭제, 역할 기반 스케일(10→19 + 디스플레이 40)로 재정의·재문서화.
  - 핀 3개 콘텐츠 모순 제거 — 마커와 1:1 정합하는 3행 메모 + 카운트 배지로 교체, 핀 마커를 .shot SVG 좌표계 안으로 이동(slice 크롭 무관하게 실제 요소 지시).
  - 설정/도움말/단축키 한 버튼 통합을 키보드·톱니 2진입점으로 분리, 대비 AAA 상향(태그라인 7.6:1), 기록 3행 캡처 종류별 현실 제목.
- 픽스한 흠: 타입 스케일 불일치 · 핀 상태 모순 · 버튼 통합 · 핀 마커 무의미 좌표 · AA 한계선 대비. (잔존 흠 없음, 노드 파서로 section 5/5·svg 34/34·핀 3=마커 3 검증)

### 3. 구미 토이 — craft 8 · 픽스 ✔
- 세부: 타이포 9 / 간격 9 / 아이콘 7 / 상태 8 / 깊이·질감 9 / 콘텐츠 9 / 시그니처 9 / polish 7
- 젤리 사탕 택타일 정체성을 토큰화(--fs/--s/--lift/--gloss)로 일관 구현. 회색 빈 박스 0(모든 빈 상태가 색조 배경+아이콘+텍스트), 썸네일은 실제같은 미니 브라우저/코드/문서 목업 SVG, 측정된 대비 주석.
- 핵심 개선:
  - 헤더 아이콘버튼 라벨 분리 — 톱니='설정', 물음표='도움말: 단축키 안내'로 aria-label·title을 아이콘 의미와 일치(스크린리더 구분 가능).
  - 줌 버튼 title 모순 수정(축소/확대 일치), 공유 토글 OFF의 ×(교차선)을 중립 점으로 교체(취소/삭제 오인 제거).
  - 업로드중+완료 모순 상태 제거, 줌 어포던스 3중 중복 정리(인-줌/원본 열기/팬으로 분리), 손댄 라인 아이콘 stroke 토큰 단일소스화.
- 픽스한 흠: 라벨 오배치·중복 · 줌 title 모순 · × 노브 · 업로드중+완료 모순 상태 · 줌 3중 중복 · placeholder 모호. (잔존 흠 없음, 검증: 깨진 var() 속성 0·충돌 데모상태 0·× 노브 0)

### 4. 와비사비 먹·낙관 — craft 7 · 픽스 ✔
- 세부: 타이포 9 / 간격 9 / 아이콘 8 / 상태 4 / 깊이·질감 8 / 콘텐츠 9 / 시그니처 9 / polish 5
- 화지 미색 + 따뜻한 먹빛 + 단일 인주 액센트의 절제·여백 craft. 1.20 모듈러 스케일·tabular-nums·한자 인장 칩, 콘텐츠 리얼리즘(회색 빈 박스를 미니 UI 목업 SVG로 박멸), seal/dot/brush 3모티프 전 섹션 적용. 정적 마감은 상위권이나 **상태(states)**에서 무너졌던 것이 가장 큰 흠이었다.
- 핵심 개선:
  - 공유 섹션 상태 모델 붕괴 해소 — data-state(idle/busy/done) 단일 진실원(SoT)으로 버튼/진행/결과가 상태당 단 하나만 노출되게 재구성.
  - 멈춘 '업로드 중' 스피너에 .spin keyframe(토큰 --t-spin:900ms) 부여, 58%에 얼어붙은 진행바를 --progress 8→92→100% 실제 애니메이션으로 교체.
  - 줌 버튼 라벨↔글리프 일치, reduced-motion 시 busy 단계 건너뛰고 즉시 done 전이, share-status에 role=status 부여.
- 픽스한 흠: idle+업로드중+완료 동시 노출(상태 모델 붕괴) · 회전하지 않는 스피너 · 얼어붙은 진행바 · 줌 라벨↔글리프 불일치. **이 3개 픽스로 craft 9 권역 진입**이 비평의 평가였고, 본 패스에서 반영했다. 위 세부점수표는 픽스 전 기준이다.
