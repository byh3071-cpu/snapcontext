날짜: 2026-07-23

## 이 Phase 가 있는 이유

P5 가 사용자에게 보관 기간 선택(1/7/30일)을 열었지만, 그 UI 를 받쳐야 할 두 축이 아직 7일 고정 세계에 있었다: 공개 개인정보처리방침(PRIVACY.md)과 스토어 카피. 게다가 per-user 토큰(0.4.0 간판)은 서버(P1~P2)만 있고 **사용자가 토큰을 꺼내 AI 도구에 연결할 화면이 없어** 실사용자에게 안 닿았다. P6 는 온보딩 UI 를 만들어 이 기능을 실제로 쓸 수 있게 하고, 문서·버전·스토어킷을 0.4.0 현실에 정렬한다.

## 설계 판단

**토큰 표시는 발급과 분리한다.** `getStoredToken`(신규, 읽기 전용)을 만들었다 — `ensureUserToken`(발급)을 표시에 쓰면 설정 화면을 열 때마다 `/token` 발급이 일어난다.

**패널 열 때마다 토큰 재조회** (e2e 가 잡은 버그). mount 시 1회만 읽으면, 발급이 첫 공유 시점에 lazy 로 일어나므로 "확장 열고→공유→설정 열기" 순서에서 토큰이 영영 '미발급'으로 남는다. `setOpen(true)` 에서 재조회하게 고쳤다.

**마스킹과 원문 분리 (보안).** 화면 `code` 태그엔 `sc_AAAA…BBBB` 마스킹만. 원문은 클로저 변수(`currentToken`)에만 살고 클립보드 복사에만 쓴다. 복붙 명령 pre 에는 원문이 평문으로 보이지만(사용자가 복사해야 하므로 불가피), console·에러엔 절대 안 싣는다.
