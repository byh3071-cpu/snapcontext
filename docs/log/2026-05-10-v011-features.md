# 작업 로그 — v0.1.1 기능 추가 및 통합

> 노션 핸드오프용 단일 문서. 이 문서만 읽어도 무엇을, 왜, 어떻게 구현했는지 파악되도록 정리.

## 1. 배경

v0.1.0에서 캡쳐·핀·Context Pack 생성 기본 파이프라인이 완성되었으나, 다음 기능이 부족했다:

- 캡쳐 이미지를 PNG로 복사·저장하는 수단
- 과거 캡쳐를 되돌아보는 히스토리
- 버그 리포트 외 프롬프트 유형
- 마우스 없이 캡쳐를 실행하는 키보드 단축키

## 2. 추가된 기능

### 2-1. PNG 복사 / 저장 (ImageActions)

| 항목 | 내용 |
|---|---|
| 파일 | `src/sidepanel/components/ImageActions.ts` |
| UI | PNG 복사 버튼 (Alt+Shift+P) + PNG 저장 버튼 |
| 구현 | `copyAnnotatedPngToClipboard` — `navigator.clipboard.write()`로 핀 주석 포함 PNG를 클립보드에 기록. `downloadAnnotatedPng` — `chrome.downloads.download()`로 PNG 파일 저장. |
| 특이사항 | MV3 Side Panel에서 `navigator.clipboard.write()`가 별도 권한 없이 동작 확인. |

### 2-2. 캡쳐 히스토리

| 항목 | 내용 |
|---|---|
| 파일 | `src/storage/history.ts`, `src/sidepanel/components/HistoryList.ts` |
| 저장소 | `chrome.storage.local`, 키 `captureHistory` |
| 정책 | 최대 50개, FIFO (오래된 순 삭제) |
| 데이터 | id, timestamp, url, title, captureType, 썸네일(base64), pinsCount, contextPack |
| UI | 썸네일 리스트, 클릭 시 저장된 Context Pack 로드, 스와이프/X 삭제, 전체 삭제 |

### 2-3. 프롬프트 템플릿 3종

| 템플릿 | 파일 | 용도 |
|---|---|---|
| 🐛 버그 리포트 | `prompts/templates/bug-report.md` | 증상·기대 동작·환경 정보 포함 AI 프롬프트 |
| 🔧 리팩토링 | `prompts/templates/refactor.md` | 코드 개선 요청 프롬프트 |
| 📐 레퍼런스 | `prompts/templates/reference.md` | UI 레퍼런스 분석 프롬프트 |

- Mustache-lite 엔진 (`src/context-pack/template-engine.ts`) — `{{var}}`, `{{#if}}`, `{{#each}}` 지원.
- 드롭다운 선택 → `chrome.storage.local`에 `promptTemplate` 키로 유지.
- 상세: `docs/adr/004-template-system.md`

### 2-4. 키보드 단축키

| 단축키 | 커맨드 | 동작 |
|---|---|---|
| `Alt+Shift+V` | `capture-visible` | Visible 캡쳐 + 사이드바 열기 |
| `Alt+Shift+E` | `capture-element` | Element 선택 모드 진입 + 사이드바 열기 |
| `Alt+Shift+D` | `capture-document` | Document 본문 영역 탐색 + 사이드바 열기 |
| `Alt+Shift+P` | `copy-png` | 현재 캡쳐 PNG 클립보드 복사 |

- `manifest.json`의 `commands` 필드로 등록.
- `chrome.commands.onCommand` → `handleCommand()` → `routeSidePanelMessage()` + `openSidePanelFromGesture()`.
- **주의:** `Ctrl+Alt` 조합은 Chrome이 지원하지 않으므로 `Alt+Shift` 사용.

### 2-5. App.ts 머지 통합

- 기존 모드 핍(visible / element / document 핍 표시) UI 제거 → ContextPackPanel 드롭다운으로 통합.
- `CAPTURE_RESULT` 수신 시 자동으로 히스토리 저장 + Context Pack 자동 생성.
- `COPY_PNG_COMMAND` 메시지 수신 → `ImageActions.copyPng()` 호출.

### 2-6. 레거시 제거

- 하드코딩 `buildPrompt()` 함수 완전 삭제.
- 관련 테스트(`tests/context-pack.test.ts`) 재작성 — 템플릿 기반 `buildTemplatePrompt` 검증으로 교체.
- 모드 핍 CSS 제거.

## 3. 버그 수정 (세션 중)

### 3-1. Alt+Shift+V 사이드바 미열림

- **증상:** `capture-visible` 커맨드로 캡쳐는 되지만 사이드바가 열리지 않음. 히스토리에서만 확인 가능.
- **원인:** `handleCommand()`에서 `await getActiveWebTabForSidePanel()` 후에 `chrome.sidePanel.open()`을 호출하여 user gesture 토큰이 소실됨.
- **수정:** `chrome.commands.onCommand` 리스너에서 `tab` 파라미터를 직접 받아 `openSidePanelFromGesture()`를 **`await` 전에** 즉시 호출.
- Element 캡쳐는 사용자 클릭 시점에 새 gesture가 생기므로 문제 없었음.

| 에셋 | v0.1.0 | v0.1.1 | 변화 |

| `service-worker.ts-*.js` | ~9.8 kB | ~10.1 kB | +3.1% (handleCommand 확장) |

Side Panel JS는 레거시 프롬프트 빌더 제거로 감소, CSS는 신규 컴포넌트(ImageActions, HistoryList, ShortcutsHelp)로 증가.

## 6. 핵심 학습 포인트

- `chrome.commands`는 `Ctrl+Alt` 조합을 지원하지 않는다. `Alt+Shift`로 변경 필요.
- `navigator.clipboard.write()`는 MV3 Side Panel에서 권한 추가 없이 동작한다.
- `chrome.sidePanel.open()`은 user gesture 컨텍스트 내에서 호출해야 한다. `await` 이후에는 gesture 토큰이 소실되어 실패한다.
- Vite `?raw` import로 `.md` 파일을 문자열로 번들에 포함하면, 런타임 파일 로드 없이 템플릿 사용 가능.
