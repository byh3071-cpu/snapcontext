## A. 카피 (한국어, 합니다체) — v2, 경쟁사 리서치 반영

적용 원칙 (리서치 권고 7): 태그라인=결과+무계정 2요소 / 훅에 대표 단축키 1개(GoFullPage) /
프라이버시 블록 상단 1/3(FireShot) / "이렇게 동작합니다" 3단계(Scribe) / 유스케이스를 기능보다
앞에 / 단축키 5개 전체 표(차별 섹션 — 5사 중 제공처 없음) / 클로징 피드백 유도.
금지: 키워드 스터핑·체인지로그 덤프·검증 불가 수치.

### ① 짧은 설명 (≤132자, 평문, 최상급·경쟁사 언급 금지)
> 캡처 한 번이면 AI에 붙여넣을 컨텍스트(프롬프트)가 자동 완성됩니다. 영역·요소·문서·풀페이지 캡처, 로그인 없는 7일 익명 공유까지.

*(결과 선언 + 무계정 신뢰 — 제출 직전 글자 수 132 이내 실측 확인)*

### ② 상세 설명 (≤16,000자 / Chrome은 이모지 OK, 웨일은 평문)
```text
화면을 캡처하는 순간, AI에 그대로 붙여넣을 컨텍스트(프롬프트)가 완성됩니다.
아이콘을 클릭하거나 Alt+Shift+V를 누르면 영역 캡처가 시작되고, 캡처와 동시에
화면·핀 주석·출처 정보가 프롬프트로 정리됩니다. 설명을 다시 쓸 필요가 없습니다.

✨ 이렇게 동작합니다
1️⃣ 캡처 — 영역·요소·문서·풀페이지 중 원하는 방식으로
2️⃣ 컨텍스트 생성 — 캡처 + 핀 주석 + 출처 메타데이터가 Context Pack(프롬프트)으로 자동 완성
3️⃣ 전달 — ChatGPT·Claude에 붙여넣거나, 7일 익명 링크로 공유

🔒 프라이버시 먼저
캡처는 기기 안에서 처리됩니다. 공유 링크를 만들 때만 업로드되며, 7일 후 접근이
차단되고 서버에서도 자동 영구 삭제됩니다. 계정도, 로그인도, 추적도 없습니다.
'컨텍스트 포함'은 기본 꺼짐 + 최초 1회 동의제 — 토큰·쿠키·로컬 데이터는 전송하지
않습니다. 추가 권한 요청 없음 · Manifest V3 · 누출 차단 E2E 검증.

💡 이럴 때 씁니다
· 버그 화면을 설명 없이 AI에게 그대로 던질 때
· 디자인 피드백을 화면 컨텍스트째 익명 링크로 전달할 때
· 긴 문서·페이지를 출처와 함께 한 장으로 남길 때

📸 캡처 4모드 + AI 컨텍스트
· 영역 선택 / 요소 선택 / 문서 본문 / 풀페이지 스크롤 전체
· 캡처 위에 핀을 찍고 메모하면 주석까지 프롬프트에 포함됩니다
· 버그 리포트 · 리팩토링 · 레퍼런스 템플릿 제공

⌨️ 단축키
Alt+Shift+V — 영역 캡처
Alt+Shift+E — 요소 캡처
Alt+Shift+M — 문서 본문 캡처
Alt+Shift+G — 풀페이지 캡처
Alt+Shift+P — AI 프롬프트 생성

💬 의견을 들려주세요
문제가 있거나 바라는 기능이 있다면 리뷰보다 먼저 알려주세요. 다음 업데이트에
반영하겠습니다. → github.com/byh3071-cpu/SnapContext/issues
```

**웨일 변환 규칙**: 위 본문에서 이모지 헤더(✨🔒💡📸⌨️💬)를 평문 헤더("[이렇게 동작합니다]" 등)로
치환, 숫자 이모지(1️⃣)는 "1." 로. 구조·문장은 동일 유지.

## B. 스크린샷 (1280×800 PNG, 풀블리드·알파 금지)
generator(`scripts/generate-store-screenshots.mjs`)가 실 dist 패널을 캡처 → **최종 UI 확정 후 재실행하면 새 디자인 자동 반영.**

| # | 장면 | Chrome | 웨일 |
|---|---|---|---|
| 1 | 영역 캡처 → AI 컨텍스트 생성 흐름 (후킹, 첫 장) | ✅ | ✅ |
| 2 | 캡처 4모드 (영역·요소·문서·풀페이지) | ✅ | ✅ |
| 3 | AI 프롬프트 자동 생성 결과 (Context Pack) | ✅ | ✅ |
| 4 | **익명 공유 링크 + 뷰어 (신규 scene)** | ✅ | ✅ |
| 5 | 단축키 안내 | ✅ | — (웨일 최대 4장) |

**generator 수정점**: ④ 공유 scene 신규 추가 · 기존 카피 "v0.1.3"→"v0.2.0" · 헤드라인 AI 컨텍스트 강조로 재배열.

## C. 프로모 타일 (텍스트 최소·고채도·다크 브랜드)
- 소형 **440×280** (필수): 코랄 글로우 다크 네이비 + 아이콘 + "SnapContext" 워드마크 + 짧은 태그라인.

- 공식 가이드: 텍스트 회피, 채도 높게, 흰/밝은회색 과다 금지, 가장자리 선명, 절반 크기로 줄여도 인식.
- generator와 같은 HTML→Playwright 캡처 방식으로 스크립트 1개 추가.

## D. 제출 메타
- **개인정보처리방침**: `docs/PRIVACY.md` **v0.2.0 갱신 완료(2026-06-11)** — 공유 업로드(R2)·7일 자동 삭제·컨텍스트 동의제 반영. 기존 "외부 전송 없음" 문구는 v0.2.0과 모순이었음(리젝 사유). 제출 URL은 master의 PRIVACY.md → **제출 전 release 브랜치 merge 필수** (안 하면 스토어가 구버전 정책을 가리킴).
- **Chrome Privacy practices 탭**: v0.1.3 신고는 "데이터 미수집" 기준 — v0.2.0은 공유 시 사용자 콘텐츠(이미지·선택적 메타데이터) 전송이 생겼으므로 대시보드 신고 항목 재검토 ("User activity/Website content" 해당 여부 체크, "판매 안 함·목적 외 사용 안 함" 서약 유지).
- **Worker 실서버 확인**: 제출 직전 생존 체크 — `GET /s/<없는id>` → 410 응답이면 정상 (2026-06-11 확인 완료). E2E upload-share는 fetch mock이라 실서버를 안 침.
- **권한 정당화**: 신규 권한 0 (0.1.3과 동일) — 리스팅에 "추가 권한 없음" 명시. (manifest diff = version 한 줄만, 검증됨)
- **웨일 차이**: 설명 **평문만**(HTML/Markdown 불가, 이모지는 텍스트로 OK), 스크린샷 **최대 4장**, 분류·공개설정·성인콘텐츠 체크 필수.
- 카테고리: 생산성(Productivity).
- 아이콘: 128×128 PNG (96 아트 + 16 패딩), 다크/라이트 배경 가독.
