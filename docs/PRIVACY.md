# SnapContext 개인정보처리방침

최종 수정일: 2026-07-23 (v0.4.0 — 보관 기간 선택·소유자 격리(owner) 반영)

공개 URL (Chrome Web Store / Whale Store 제출용):

https://github.com/byh3071-cpu/SnapContext/blob/master/docs/PRIVACY.md

## 수집하는 정보

SnapContext는 계정, 로그인, 식별자, 사용 추적 등 어떠한 개인정보도 수집하지 않습니다.

## 데이터 저장

- 캡처 이미지, 핀 주석, 설정값은 사용자의 브라우저 로컬 저장소(`chrome.storage.local`)에만 저장됩니다.
- 사용자가 직접 실행하는 **공유 기능을 제외하면**, 외부 서버로 전송되는 데이터는 없습니다.

## 공유 기능 (v0.2.0)

- 공유는 사용자가 '공유 링크' 버튼을 눌렀을 때에만 동작합니다. 자동 업로드는 없습니다.
- 공유 시 해당 캡처 이미지가 익명으로 서버(Cloudflare Workers + R2)에 업로드되고, 링크를 아는 사람만 볼 수 있는 공유 URL이 생성됩니다.
- '컨텍스트 포함' 옵션을 켠 경우에 한해 캡처의 컨텍스트 텍스트(핀 메모, 페이지 제목·URL 등 메타데이터)가 함께 업로드됩니다. 이 옵션은 **기본 꺼짐**이며, 최초 사용 시 1회 동의를 받습니다.
- 사용자의 브라우저 쿠키·로그인 세션·다른 사이트의 인증 정보·디버그 로그는 어떤 경우에도 전송되지 않습니다 (자동 테스트로 검증). 단, 공유 업로드 시 SnapContext가 발급한 개인 토큰(소유자 식별용)은 `Authorization` 헤더로 전송될 수 있으며, 이는 아래 '소유자 격리' 절을 따릅니다. 이 토큰은 사용자를 위해 발급된 것이지 다른 서비스의 인증 정보가 아닙니다.
- 업로드된 데이터는 **사용자가 선택한 보관 기간(1·7·30일, 기본 7일)이 지나면 접근이 차단되고 서버 저장소에서 자동 영구 삭제**됩니다. 보관 기간은 확장 설정에서 고를 수 있으며, 기본값은 7일입니다.
- 업로드에 계정·이메일·실명 등 신원 정보는 결부되지 않습니다. 단, 아래 '소유자 격리' 절에 따라 사용자 개인 토큰의 단방향 해시가 결부될 수 있으며, 이는 신원을 식별하지 않습니다. 공유 데이터는 광고·분석 등 다른 목적으로 사용하지 않습니다.

## 소유자 격리 (owner · v0.4.0)

- AI 코딩 도구(MCP)에서 자기 업로드만 조회할 수 있도록, 공유 업로드 시 사용자 개인 토큰의 **단방향 해시(SHA-256)**가 캡처에 결부됩니다. 이 값을 'owner'라고 부릅니다.
- owner는 **계정·이메일·실명이 아닌** 무작위 토큰에서 파생된 값이며, 토큰에서 되돌려 신원을 알아낼 수 없는 익명 식별자(pseudonymous)입니다. 서버는 전송받은 토큰으로 owner 해시를 계산할 뿐, **토큰 원문을 저장하지 않습니다**(전송과 저장은 별개입니다).
- 개인 토큰이 없는 업로드는 owner 없이(익명으로) 저장되며, 이 경우 자기 조회 격리 없이 링크를 아는 사람만 접근하는 기존 방식과 동일합니다.

## AI 코딩 도구 연동 (v0.3.0)

- 사용자가 공유 업로드한 캡처는, 사용자 본인이 발급·보관하는 개인 토큰을 설정한 AI 코딩 도구(Claude Code·Cursor 등)에서 조회할 수 있습니다(MCP).
- 이 기능은 **새로운 데이터를 수집하지 않습니다** — 조회 대상은 위 '공유 기능' 절에 따라 사용자가 직접 업로드한 데이터뿐이며, 사용자가 선택한 보관 기간(1·7·30일) 만료 정책이 동일하게 적용됩니다.
- **MCP 조회 요청**은 유효한 개인 토큰이 없으면 서버가 거부합니다(익명 공유 업로드 자체는 토큰 없이도 가능하며, 이 경우 소유자 격리 없이 저장됩니다). 토큰은 확장 프로그램이나 스토어를 통해 배포되지 않으며, 사용자가 직접 발급·보관합니다.

## 권한 사용 목적

| 권한 | 목적 |
|---|---|
| sidePanel | 캡처 도구 UI를 사이드 패널에 표시하기 위해 사용 |
| storage | 설정값과 캡처 히스토리를 로컬에 저장하기 위해 사용 |
| scripting | 페이지에 콘텐츠 스크립트를 주입하여 요소 선택·본문 영역 탐색을 수행하기 위해 사용 |
| downloads | PNG 파일을 저장하기 위해 사용 |
| tabs | 현재 탭의 URL·제목 등 메타데이터를 읽기 위해 사용 |
| windows | 캡처 대상 창을 식별하기 위해 사용 |
| host_permissions (`<all_urls>`) | 모든 웹 페이지에서 화면 캡처 및 콘텐츠 스크립트 실행을 허용하기 위해 사용 |

## 제3자 공유

사용자 데이터를 판매·프로파일링·광고·분석 목적으로 제3자와 공유하지 않습니다.
단, 사용자가 직접 '공유 링크'를 생성해 전달한 경우에 한해, 해당 링크를 아는
사람이 공유된 캡처(및 사용자가 동의해 포함한 컨텍스트)에 접근할 수 있습니다.
이는 위 '공유 기능' 절에 기술된 사용자 주도 동작이며, 사용자가 선택한 보관
기간(1·7·30일) 후 접근 차단·영구 삭제 정책이 동일하게 적용됩니다.
