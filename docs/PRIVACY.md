# SnapContext 개인정보처리방침

최종 수정일: 2026-06-11 (v0.2.0 — 익명 공유 기능 반영)

공개 URL (Chrome Web Store / Whale Store 제출용):

https://github.com/byh3071-cpu/SnapContext/blob/master/docs/PRIVACY.md

## 수집하는 정보

SnapContext는 계정, 로그인, 식별자, 사용 추적 등 어떠한 개인정보도 수집하지 않습니다.

## 데이터 저장

- 캡처 이미지, 핀 주석, 설정값은 사용자의 브라우저 로컬 저장소(`chrome.storage.local`)에만 저장됩니다.
- 사용자가 직접 실행하는 **공유 기능을 제외하면**, 외부 서버로 전송되는 데이터는 없습니다.

## 공유 기능 (v0.2.0)

- 공유는 사용자가 '공유 링크' 버튼을 눌렀을 때에만 동작합니다. 자동 업로드는 없습니다.
- 공유 시 해당 캡처 이미지가 익명으로 서버(Cloudflare Workers + R2)에 업로드되고, 링크를 아는 사람만 볼 수 있는 공유 URL이 생성됩니다.
- '컨텍스트 포함' 옵션을 켠 경우에 한해 캡처의 컨텍스트 텍스트(핀 메모, 페이지 제목·URL 등 메타데이터)가 함께 업로드됩니다. 이 옵션은 **기본 꺼짐**이며, 최초 사용 시 1회 동의를 받습니다.
- 토큰, 쿠키, 브라우저 로컬 데이터, 디버그 로그는 어떤 경우에도 전송되지 않습니다 (자동 테스트로 검증).
- 업로드된 데이터는 **7일 후 접근이 차단되고 서버 저장소에서 자동 영구 삭제**됩니다.
- 업로드에 계정·식별자가 결부되지 않으며, 공유 데이터는 광고·분석 등 다른 목적으로 사용하지 않습니다.

## 권한 사용 목적

| 권한 | 목적 |
|---|---|
| sidePanel | 캡처 도구 UI를 사이드 패널에 표시하기 위해 사용 |
| storage | 설정값과 캡처 히스토리를 로컬에 저장하기 위해 사용 |
| scripting | 페이지에 콘텐츠 스크립트를 주입하여 요소 선택·본문 영역 탐색을 수행하기 위해 사용 |
| downloads | PNG 파일을 저장하기 위해 사용 |
| tabs | 현재 탭의 URL·제목 등 메타데이터를 읽기 위해 사용 |
| windows | 캡처 대상 창을 식별하기 위해 사용 |
| host_permissions (`<all_urls>`) | 모든 웹 페이지에서 화면 캡처 및 콘텐츠 스크립트 실행을 허용하기 위해 사용 |

## 제3자 공유

사용자 데이터를 판매·프로파일링·광고·분석 목적으로 제3자와 공유하지 않습니다.
단, 사용자가 직접 '공유 링크'를 생성해 전달한 경우에 한해, 해당 링크를 아는
사람이 공유된 캡처(및 사용자가 동의해 포함한 컨텍스트)에 접근할 수 있습니다.
이는 위 '공유 기능' 절에 기술된 사용자 주도 동작이며, 7일 후 접근 차단·영구
삭제 정책이 동일하게 적용됩니다.
