## 리서치 트랙 R — snap_capture 원격 캡처 (Web Push PoC)

> 0.4.0 구현 스코프 **밖** — 독립 리서치. 결과가 go면 0.5+ 트랙으로 승격.

- 배경: 0.3.0 Phase 0 결론 = 드랍 (원격 MCP→확장 즉시 깨우기의 검증된 패턴 부족, Web Push가 최유망하나 완결 E2E 공개 사례 미발견). 근거: `docs/research/phase0-capture-trigger.md`.
- **재검토 3조건** (phase0-capture-trigger.md:99 — 전부 충족해야 go):
  1. Web Push silent push + subscription 수명 관리 PoC 성공
  2. MCP 툴이 비동기 캡처 결과를 ≤N초 내 반환하는 프로토콜 확정
  3. 탭 선택·브라우저 오프라인 시 에러 시맨틱 ADR
- 산출물: `docs/research/webpush-capture-poc.md` + go/no-go 판정. PoC 코드는 별도 브랜치(머지 안 함).

## 에이전트 역할 매핑

> SoT = `yohan-brain/memory/core/agent-roster.yaml` (실제 모델·티어는 그 파일이 결정 — 여기 복제 금지). 전 Phase 일괄 최상위 티어 상시 할당 금지.

| lifecycle | 역할군 | 적용 |
|-----------|--------|------|
| implement | balanced_or_strong | P1~P6 구현 |
| verify | balanced_or_strong (기본) → 최종 릴리즈 verify만 strong 상향 | 각 Phase 검증 |
| review | strong_or_conductor | P2 인증·격리, P6 PRIVACY 정합 — 리뷰 필수 |
| research | cheap_or_balanced | 트랙 R PoC 조사 |
| release | conductor + **사람 게이트(secret·lifecycle·deploy·스토어·태그·머지)** | 운영 태스크 전부 |
