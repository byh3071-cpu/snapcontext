# SnapContext 로드맵

> 대상 명세: [PRD 0.3.0](./PRD-0.3.0.md). 이 문서는 **실행 순서·완료 기준·에이전트 매핑**만 담는다(제품 스펙 중복 금지).

3트랙 구조. 트랙 A(0.2.x 잔여)와 트랙 B(0.3.0 MCP)는 **스코프상 독립**이나, 병렬 실행에는 격리 규칙이 붙는다:

> **병렬 실행 규칙**: 트랙 A·B를 동시에 굴리려면 **worktree 분리 필수**(트랙별 독립 `mission.json`). roster concurrency 정책상 `same_repo_same_branch`는 forbidden — 같은 checkout에서 두 트랙의 mission을 겹치면 안 된다. 같은 checkout 하나로 갈 거면 **직렬화**: 트랙 A 완결 → Phase 0에서 mission 재설정 → 트랙 B 착수.

## 트랙 B — 0.3.0 MCP (본 트랙)

### Phase 0 — 리서치 완결·기술 결정

- **목표**: PRD 미해결 질문 A~F에 출처·날짜 붙은 답을 확정하고, 착수 전 스코프를 고정.
- **완료 기준**:
  - 질문 A~F 결론 확정(A/B/C = P0 필수). 1년 이상 자료는 현행성 재확인.
  - `snap_capture` **MVP 포함/드랍 판단** 확정(질문 C 근거 기반).
  - 인덱스 저장소 택1(질문 D), 최소 인증 방식 택1(질문 E).
  - `.vhk/mission.json` **재설정** — 현재 v0.2 UI용 stale(scope=`src/sidepanel`, forbidden=`src/background`). 0.3.0 스코프(worker MCP·수집)로 갱신.
- **산출물**: ADR(transport·인덱스 저장소·인증), 갱신된 `mission.json`, 확정된 툴 목록.
- **에이전트**: research(cheap_or_balanced) 조사 → product-spec(conductor_or_strong) 결정·ADR.

### Phase 1 — Worker MCP 서버 뼈대 + read-only 2툴

- **목표**: 기존 R2 데이터를 대상으로 `snap_history`·`snap_pack`을 read-only로 세운다.
- **완료 기준**:
  - Worker에 MCP 라우트 추가(기존 `/upload`·`/i`·`/s` 공유 라우트와 공존).
  - **`snap_history` 데이터 소스는 Phase 0 질문 D 결정에 종속** — (a) `index.json` 택 시: 기존 R2 `{id}.json` 객체를 `list()`로 열거해 인덱스를 초기 구축하는 **backfill 스크립트를 Phase 1에 포함**, (b) R2 `list()` 직접 열거 택 시: backfill 불요. (→ 목표문 "기존 R2 데이터 대상"과 완료기준의 저장소 표현을 일치시킴)
  - `id`로 R2 원본 조회 → `snap_pack` 동작. 만료/없는 id = 명시적 에러.
  - worker vitest green.
- **산출물**: MCP 라우트 코드, 인덱스 스키마 확정판, (index.json 택 시) backfill 스크립트, 툴 단위 테스트.
- **에이전트**: implement(balanced_or_strong), verify(balanced_or_strong).

### Phase 2 — 확장→Worker 수집 파이프라인 + 최소 인증

- **목표**: 확장이 캡처를 인덱스에 등록하고, "내 데이터만" 조회하도록 최소 인증을 건다.
- **완료 기준**:
  - 확장 캡처 → R2/인덱스 반영 경로 구현(질문 E 결정: 직접 PUT vs Worker 경유).
  - 수집 시 인덱스 엔트리 자동 갱신. 필요 시 `snap_pack` 반환 품질을 위해 저장 범위 확장(핀 좌표·노트 등).
  - **저장 범위 확장 필드는 기존 사용자 동의 범위 내 + 공개 `/s` 뷰어에 신규 필드 미노출 확인**(누출 회귀).
  - **worker secret 프로비저닝**(`wrangler secret put` — **사람 게이트**)으로 bearer 토큰 주입 → 조회 라우트 보호(토큰 없으면 미공개).
  - `src/utils/upload.ts` 재사용/확장, 누출 회귀 테스트 유지.
- **산출물**: 수집 클라이언트 변경, 인증 미들웨어, 회귀 테스트.
- **에이전트**: implement(balanced_or_strong), review(strong_or_conductor) — 인증·누출은 리뷰 필수.

### Phase 3 — snap_analyze (+ snap_capture 조건부)

- **목표**: 파생 툴 `snap_analyze` 추가. `snap_capture`는 Phase 0 판단 결과에 따름.
- **완료 기준**:
  - `snap_analyze` — `snap_pack` 위에 요약/이슈 추출 반환(분석 위치 = Worker vs 클라이언트 에이전트, Phase 0/3 결정).
  - `snap_capture` = Phase 0에서 "포함"이면 원격 트리거 패턴 구현, "드랍"이면 0.4+로 이월(스킵).
  - **worker vitest green(신규 툴 테스트 포함)**.
- **산출물**: `snap_analyze` 코드·테스트, (조건부) capture 트리거 코드.
- **에이전트**: implement(balanced_or_strong), verify(balanced_or_strong).

### Phase 4 — DoD 검증 + 배포 + 문서화 + 태그

- **목표**: 배포본을 대상으로 DoD를 실 클라이언트로 검증하고 릴리즈.
- **완료 기준**:
  - **worker 배포**(`wrangler deploy` — **사람 게이트**) → **배포본 대상 E2E**.
  - Claude Code **또는** Cursor에서 원격 MCP 등록 → `snap_history`·`snap_pack` 실호출 E2E 성공(PRD DoD 1~2).
  - **DoD 3 (negative test)**: 만료/없는 id 실호출 시 조용한 빈 반환이 아니라 **명시적 에러 반환 확인**.
  - `pnpm test`(root+worker) green, `tsc --noEmit`+`vite build`+worker 빌드 통과, `vhk mission check` 위반 0(PRD DoD 4).
  - ADR·`docs/changelog.md` 0.3.0 기록, `v0.3.0` 태그(사람 게이트).
- **산출물**: E2E 검증 기록, changelog 0.3.0, git 태그.
- **에이전트**: verify(strong_or_conductor) — 최종 verify만 strong 상향, release(conductor + 사람 게이트).
